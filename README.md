# Design and Development of an NLP-Based Data Pipeline for Fishing Activity Analysis

This repository contains the notebooks developed for the Bachelor's thesis (Trabajo Fin de Grado)
**"Design and Development of an NLP-Based Data Pipeline for Fishing Activity Analysis"**, carried
out in the *Grado en Ingeniería y Sistemas de Datos* at the **Universidad Politécnica de Madrid
(ETSIT – UPM)**.

- **Author:** Juan José López Rodrigo
- **Tutor:** Sergio Muñoz López
- **Department:** Departamento de Ingeniería de Sistemas Telemáticos · Grupo de Sistemas Inteligentes
- **Date:** June 2026

The implementation is applied to the **Valmayor reservoir** case study. The pipeline transforms
public, weakly structured digital evidence from YouTube into a structured quarterly dataset, combines
it with open environmental variables, and uses it for exploratory classification and scenario analysis.

> **Scope note.** The project is exploratory and methodological. The target is a *proxy* of
> *documented* fishing activity derived from public text and LLM-assisted extraction. It is **not** a
> measurement of real fishing effort, catches, fish abundance or ecological status, and the models are
> not operational forecasting tools.

## Pipeline overview

The notebooks are intended to be **executed in numerical order**. Each notebook performs one stage of
the workflow and writes intermediate files that are consumed by the next stage:

| Notebook | Stage |
| --- | --- |
| `01_youtube_scraping.ipynb` | YouTube data collection (public metadata and transcripts) |
| `02_dataset_preparation.ipynb` | Cleaning and consolidation of the raw video dataset |
| `03_candidate_selection.ipynb` | Candidate filtering and priority scoring |
| `04_candidate_review.ipynb` | Review and consolidation of the candidate set |
| `05_llm_extraction.ipynb` | LLM-assisted structured information extraction |
| `06_llm_video_analysis.ipynb` | Aggregation and analysis of the extracted annotations |
| `07_activity_target_construction.ipynb` | Construction of the quarterly activity target |
| `08_weather_features.ipynb` | Meteorological feature integration (Open-Meteo) |
| `09_hydrology_features.ipynb` | Hydrological / reservoir feature integration (MITECO) |
| `10_pressure_features.ipynb` | Atmospheric-pressure feature integration |
| `11_modeling_dataset.ipynb` | Assembly of the unified quarterly modelling dataset |
| `12_classification_modeling.ipynb` | Classification modelling and temporal validation |
| `13_scenario_analysis.ipynb` | Exploratory scenario / what-if analysis |

In short, the pipeline covers **YouTube data collection, candidate filtering, LLM-assisted extraction,
activity target construction, environmental feature integration, classification modelling and scenario
analysis.**

An auxiliary water-quality exploration notebook that is not part of the main pipeline is kept separately
in `notebooks_extra/`.

## Environment and configuration

The notebooks were originally developed in **Google Colab**, using a Google Drive project folder as the working 
directory. In the released repository, the project root is named:

`tfg-fishing-activity-nlp-pipeline`

When running the notebooks in another environment, the `BASE_DIR` variable at the beginning of each notebook 
should be adapted to the local or Google Drive path where the repository has been placed.

Some stored notebook outputs may still show the original development path used during execution. These outputs 
have been preserved to keep the documented runs unchanged.

## Data

Some large or generated artefacts are excluded from version control via `.gitignore` (see that file for
details), including the raw MITECO Access database (`.mdb`), compressed archives and model binaries.
These can be regenerated by running the corresponding notebooks or obtained from their original public
sources (YouTube; Open-Meteo; MITECO reservoir data).

## Requirements

See `requirements.txt`. The notebooks rely on the standard scientific Python stack plus a few
data-collection and parsing libraries. The `google.colab` module is provided by the Colab runtime and
is not installable via `pip`.
